คู่มือตั้งค่า SMEC Online (Google Sheets + Drive)

ครั้งแรกใช้เวลาประมาณ 10–15 นาที — ต้องตั้ง 3 อย่าง: OAuth Client ID, Spreadsheet ID, และ Drive Folder ID
หัวข้อ
  • ส่วนที่ 1: สร้าง Google Cloud Project + เปิด API
  • ส่วนที่ 2: ตั้งค่า OAuth Consent Screen
  • ส่วนที่ 3: สร้าง OAuth Client ID
  • ส่วนที่ 4: เตรียม Spreadsheet (นำเข้าจาก xlsx)
  • ส่วนที่ 5: สร้าง Drive Folder สำหรับเอกสารแนบ
  • ส่วนที่ 6: แก้ไข config.js + รัน Local Server
  • ส่วนที่ 7: Deploy ขึ้น Online (ถ้าต้องการ)
  • 🔧 Troubleshoot

ส่วนที่ 1: สร้าง Google Cloud Project + เปิด API

1 เปิด Google Cloud Console แล้ว Sign in ด้วย Google account ที่จะใช้
2 ที่มุมบนซ้ายข้างคำว่า "Google Cloud" คลิก Select a project → NEW PROJECT
  • ตั้งชื่อ Project: เช่น SMEC-JobDB
  • คลิก CREATE
3 เปิด Google Sheets API:
  • ไปที่ APIs Library — Google Sheets API
  • คลิก ENABLE
4 เปิด Google Drive API:
  • ไปที่ APIs Library — Google Drive API
  • คลิก ENABLE

ส่วนที่ 2: ตั้งค่า OAuth Consent Screen

1 ไปที่ OAuth consent screen
2 เลือก User Type:
  • ถ้าเป็น account ส่วนตัว (gmail.com) → เลือก External
  • ถ้าเป็น Google Workspace ขององค์กร → เลือก Internal (ง่ายกว่า ไม่ต้องผ่าน verification)
คลิก CREATE
3 กรอกข้อมูล App information:
  • App name: SMEC Job Database
  • User support email: อีเมลของคุณ
  • Developer contact: อีเมลของคุณ
ที่เหลือกด SAVE AND CONTINUE ผ่านไปทุกหน้า
4 (เฉพาะ External) หน้า "Test users" — เพิ่มอีเมลของคุณและคนที่จะใช้แอป (ถ้ายังไม่ verify จะอนุญาตเฉพาะ test users เท่านั้น) แล้วกด SAVE AND CONTINUE

ส่วนที่ 3: สร้าง OAuth Client ID

1 ไปที่ Credentials → คลิก + CREATE CREDENTIALS → OAuth client ID
2 Application type: เลือก Web application
3 Name: ใส่อะไรก็ได้ เช่น SMEC Web Client
4 Authorized JavaScript origins — สำคัญมาก!
กด + ADD URI แล้วใส่ URL ที่จะรันแอปนี้:
กรณีใช้ใส่อะไร
ทดสอบ Local (ผ่าน start.bat)http://localhost:8765
Deploy บน GitHub Pageshttps://[username].github.io
Deploy บน Vercelhttps://[your-app].vercel.app
Domain เองhttps://yourdomain.com
เพิ่มได้หลายอันถ้าจะใช้หลาย URL
5 คลิก CREATE → จะมี popup แสดง Client ID

คัดลอก Client ID เก็บไว้ — จะใช้ใน config.js
ตัวอย่าง: 1234567890-abcdefghijk.apps.googleusercontent.com

ส่วนที่ 4: เตรียม Spreadsheet

1 ไปที่ Google Sheets สร้าง Spreadsheet ใหม่ ตั้งชื่อ SMEC Job Database
2 เปิด Spreadsheet นั้น → File → Import
  • เลือก Upload
  • ลาก SMEC_Job_Database.xlsx เข้ามา (จากโฟลเดอร์ D:\Document\SMECDB\)
  • Import location: เลือก Replace spreadsheet
  • คลิก Import data
3 หลัง Import แล้ว ตรวจสอบว่ามี 5 ชีต: ชีต1, ใบส่งของชั่วคราว, GatePass, Schedule_DB, Costs
4 คัดลอก Spreadsheet ID จาก URL ของ Spreadsheet: 
https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz0123/edit
ส่วนที่ขีดเส้นเหลืองคือ Spreadsheet ID — เก็บไว้

ส่วนที่ 5: สร้าง Drive Folder สำหรับเอกสารแนบ

1 ไปที่ Google Drive → คลิก + New → New folder → ตั้งชื่อ SMEC_Attachments
2 ดับเบิลคลิกเข้าโฟลเดอร์ที่เพิ่งสร้าง → คัดลอก Folder ID จาก URL: 
https://drive.google.com/drive/folders/1XyZ9876543abcDeFgHiJkLm
ส่วนที่ขีดเส้นเหลืองคือ Folder ID — เก็บไว้
3 (ไม่บังคับ) เก็บ Logo ไว้ใน Drive
  • อัปโหลด logo.jpg เข้าไปในโฟลเดอร์ SMEC_Attachments นี้
  • คลิกขวาที่ไฟล์ → Get link → คัดลอก URL
  • ส่วน 1abc...xyz ใน URL คือ Logo File ID
ถ้าไม่ตั้ง ระบบจะแสดง logo เป็น text badge แทน

ส่วนที่ 6: แก้ไข config.js + รัน Local Server

1 เปิดไฟล์ D:\Document\SMECDB\WebApp-Online\assets\config.js ด้วย Notepad (หรือ VS Code)
2 แก้ค่า 3 ตัวให้ตรงกับที่คัดลอกมาในส่วน 3, 4, 5: 
const CONFIG = {
  CLIENT_ID: '1234567890-abc.apps.googleusercontent.com',  // ← จากส่วน 3
  SPREADSHEET_ID: '1AbCdEfGhIjKlMnOpQrStUvWxYz0123',       // ← จากส่วน 4
  ATTACHMENTS_FOLDER_ID: '1XyZ9876543abcDeFgHiJkLm',       // ← จากส่วน 5
  LOGO_FILE_ID: '',  // ← (ไม่บังคับ) จากส่วน 5 ข้อ 3
  ...
};
Save ไฟล์
3 รัน Local server: ไปที่โฟลเดอร์ WebApp-Online ดับเบิลคลิก start.bat
เบราว์เซอร์จะเปิด http://localhost:8765/WebApp-Online/
4 คลิก เข้าสู่ระบบ Google ในแถบซ้าย
  • เลือกบัญชี Google
  • กด Continue เมื่อมีคำเตือน "Google hasn't verified this app" (ถ้าเป็น External + ยังไม่ verify)
  • อนุญาต Sheets + Drive
✅ ถ้าเห็นข้อความ "โหลดข้อมูลสำเร็จ" และมีข้อมูล 5 ชีตขึ้น → เสร็จเรียบร้อย!

ส่วนที่ 7: Deploy ขึ้น Online (ถ้าต้องการให้คนอื่นเข้าได้)

ถ้าใช้ในเครื่องเดียวก็ไม่ต้องทำส่วนนี้ แต่ถ้าจะให้ทีมเข้าจากที่ไหนก็ได้:

วิธีที่ 1: GitHub Pages (ฟรี)

  1. สร้าง GitHub repo ใหม่ เช่น smec-jobdb
  2. อัปโหลดทุกไฟล์ในโฟลเดอร์ WebApp-Online เข้า repo
  3. ใน Settings → Pages → Source: main branch, root
  4. เพิ่ม https://[username].github.io ใน Authorized origins (ส่วนที่ 3 ข้อ 4)

วิธีที่ 2: Vercel (ฟรี + ง่ายมาก)

  1. เข้า vercel.com → login ด้วย GitHub
  2. Import GitHub repo ที่อัปโหลดในวิธีที่ 1
  3. Deploy ได้ URL https://your-app.vercel.app
  4. เพิ่ม URL นี้ใน Authorized origins
⚠️ หลัง deploy แล้ว ต้องแก้ Authorized JavaScript origins ใน Google Cloud Console (ส่วนที่ 3 ข้อ 4) ให้รวม URL ใหม่ด้วย — ไม่งั้น OAuth จะ error

🔧 Troubleshoot ปัญหาที่พบบ่อย

"Error 400: redirect_uri_mismatch" หรือ "origin_mismatch"

URL ที่เปิดอยู่ไม่ตรงกับ Authorized JavaScript origins ใน Google Cloud Console — กลับไปที่ส่วน 3 ข้อ 4 เพิ่ม URL ปัจจุบันให้ตรงเป๊ะ (รวม port)

"Access blocked: This app's request is invalid"

ยังไม่ได้เปิด Sheets API หรือ Drive API — กลับไปทำส่วน 1 ข้อ 3-4

"Google hasn't verified this app"

ปกติถ้าเป็น External + Test users — กด Advanced → Go to SMEC Job Database (unsafe) ก็ใช้ได้ ถ้าจะเอาออก verification ต้องส่งให้ Google review (ใช้เวลาเป็นเดือน)

"Permission denied" หลัง sign-in

อีเมลที่ใช้ sign-in ยังไม่ได้เพิ่มใน Test users — ส่วน 2 ข้อ 4

โหลดข้อมูลไม่ขึ้น แต่ sign-in สำเร็จ

SPREADSHEET_ID ผิด หรือ Spreadsheet ไม่ได้อยู่ใน account ที่ sign-in

อัปโหลดเอกสารแนบไม่ได้

ATTACHMENTS_FOLDER_ID ผิด หรือ folder ไม่ได้อยู่ใน account ที่ sign-in
